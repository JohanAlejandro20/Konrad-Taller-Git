MVC

Es un patron arquitectónico en donde separa diferentes responsabilidades en un proyecto u aplicación. Su nombre se compone de sus siglas M(modelo), V(vista) y C(controlador) estas cumplen un rol importante en este patro.A continuación se expicará la funcionalidad de cada una de estas.

Modelo

Es el que gestina consulta y actualiza los datos en el proyecto u aplicación.
El modelo debe estar completamente independiente de la vista.
Ejemplo:
si en algun momento la aplicación la cual se esta relizando via web se requiere pasarla a una aplicacion para IOS, esto no debe afectar en lo absoluto a la vista ya que una es autónoma de la otra.

Funciones:

  • Conectarse a la base de datos
  • Reaizar alguna validacion con respecto a la base de datos
  • Conectarse con servicios web

Vista

Es la informacion que se le presenta de forma amigable al usuario para que este puede interactuar con la aplicacion de modo interactivo.
Asi como hablamos anteriormente del Modelo, la vista tiene que ser totalmente independiente del modelo.
Ejemplo:
Una interfaz de usuario.

Funciones:

  • Definir la estructura y aparencia de los datos en una pantalla

Controlador

Se encarga de la comunicacion entre el modelo y la vista y vs.

Funciones

  • Procesar la información que le entrega la vista a algo que pueda entender el modelo.
  • Procesar la información que le otorga el modelo para que la vista la pueda mostrar.
  • Llevar acabo la navegacion de una aplicacion tanto web como movil

SERVIDOR DE APLICACIONES

Es una aplicación que administra los recursos de un computador para ofrecer servicios. Permite realizar el despliegue de las aplicaciones.

Beneficios

Alta disponibilidad: 24 horas al dia 365 dias al año.
Mantenimiento: Actualizar, Depurar, Mantener. Reusabilidad y Modularidad.
Escalabilidad: Transacciones y concurrencia, Capacidad de almacenamiento.

Algunos servidores de aplicaciones en Java:

  • Glassfish
  • TomCat
  • WebLogic
  • Jetty
  • Netty
  • WildFly

REPOSITORIOS

Un repositorio, depósito o archivo es un sitio centralizado donde se almacena y mantiene información digital, habitualmente bases de datos o archivos informáticos.

Los repositorios sirven para:

  • Gestionar proyectos de forma ágil.
  • Trabajo en equipo.
  • Control de Versiones (En el tiempo).
  • Comparaciones y restauración de versiones anteriores.
  • Fusión de cambios.
  • Trabajar con distintas partes de un proyecto (Ramas).

REPOSITORIOS MAS USADOS:

  • Mercurial
  • Subversion
  • git
  • Github
